## 지원서의 질문 중 Long form question의 합격/불합격을 결정짓는 주요인이라고 나와 있어, 해당 내용만 요약해두었습니다.

- **21–22(질문 요약)**
    1. 2020–21 MIEE였는가? (Yes/No) → Yes면 커뮤니티 기여 설명
    2. 접근성: MS 도구로 다양한 학습 요구 지원
    3. 수업/프로젝트 + 21CLD 스킬 + 루브릭 점수
    4. **학습 산출물(artifact) 링크(URL)**
    5. 산출물 설명: 목표 달성/도구 기여 분석
    6. 본인 성장 + 동료 성장 지원 사례 2개
- **22–23(질문 요약)**
    1. 접근성: MS 도구로 다양한 학습 요구 지원
    2. 수업/프로젝트 + 21CLD 스킬 + 루브릭 점수
- **24-25(질문 요약)**
    
    🚩 **24-25 “신규 지원자” (기존 MIEE 아님)**
    
    1. 접근성(개인화/차별화)을 높이기 위해 ms 도구 어떻게 활용하는가— HOW 반드시 포함
    2. ms 기술 통합한 수업/프로젝트 설명(21CLD 최소 1개 포함 + 루브릭 점수 포함)
    3. **학습 산출물(artifact) URL 제공(sway,ppt,word)**
    4. 학습 목표 명시 + 3에서 제공한 산출물과 사용한 도구들이 학습목표를 달성하도록 어떻게 도왔는가
    5. 생성형 AI 활용 개인 사례 - 무엇을 배웠는가
    6. 본인 성장 + 동료 성장 지원 사례 2개
    
    🚩 **24–25 “갱신(renewal)”**
    
    1. 올해 배운 것 공유(연수/발표/콘텐츠) + 영향(impact)
    2. ms 협업 도구(동료/학습자) + 도구의 가치
    3. 동료들이 내가 어떤 도구/주제를 잘한다고 말하는가
    4. 3번에서 답한 도구/주제와 관련해 누군가를 도와주었던 경험
    5. MIEE 커뮤니티 기여(환원)
    6. 앞으로 12개월 커뮤니티 참여 3가지 계획
    7. 생성형 AI 활용 개인 사례(무엇/도움/배움)
- **25-26(질문 요약)**
    1. 일반 M365 도구 1개 선택 → 교육자 역할 지원 시나리오 설명
    2. 교육 특화 도구 1개 선택 → 교육자 역할 지원 시나리오 설명

---

**💡 지원서 문항은 ‘고정된 시험’이 아니라, 그 해 Microsoft가 교육 현장에서 중요하게 보는 역량을 반영해서 계속 수정되는 것으로 보입니다.**

- 접근성과 21CLD가 전면에 나옵니다.(2022–23)
- 산출물 링크까지 포함해 증거 기반으로 선정합니다.(2021-2022, 2024-2025)
- 자격 갱신에서는 공유의 영향과 커뮤니티 환원을 더 강하게 묻기도 합니다(2024–25).
- 최근에는 도구를 하나 고르는 방식으로 바뀌어 ‘실제 활용 깊이’를 보는 흐름도 나타났습니다.(2025–26).

그래서 올해 지원서가 어떻게 나오든, 준비는 **대표 사례 1~2개를 증거와 영향 중심으로 정리**하면 어떤 문항에도 대응할 수 있다고 생각합니다…!
